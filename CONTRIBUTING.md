# Contribuir a Heimdall Desktop

Gracias por tu interés en contribuir.

## Reglas generales

- Respeta la arquitectura existente
- No mezclar lógica de negocio con UI
- Un diálogo por fichero
- Documentar cambios relevantes

## Flujo de trabajo

1. Fork del repositorio
2. Crear rama feature/
3. Commits claros y atómicos
4. Pull Request documentado

## Estilo

- Python 3.10+
- Imports explícitos
- Manejo de errores
